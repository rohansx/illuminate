### Task R — Path normalization in `audit_with_files`

**File:** `crates/illuminate-audit/src/lib.rs`.

**Goal.** When `audit_with_files` receives an absolute file path AND the index was populated with repo-relative paths, normalize the absolute path to repo-relative before lookup.

**Approach.**
1. Add an optional `repo_root: Option<PathBuf>` field on `Auditor` (set via `with_index` extension or new `with_index_and_root` constructor — pick whichever is cleaner).
2. In `compute_impact` and `build_seed_qualifiers`, if a file path is absolute AND `repo_root` is set, strip the prefix to produce relative form before passing to `lookup_file` / building qualified names.
3. If the file path is already relative or `repo_root` is unset, pass through as today.

**Tests.** Add to `crates/illuminate-audit/tests/impact_tests.rs`:
- `audit_with_files_normalizes_absolute_paths_when_root_set` — Build auditor with `repo_root = tempdir`, pass absolute paths into `audit_with_files`, assert defined_symbols still finds them.
- `audit_with_files_passes_through_relative_paths` — No `repo_root`, relative paths still work.
- `audit_with_files_passes_through_when_root_unset` — `repo_root` unset, absolute paths produce empty (current behavior preserved when no root).

**CLI/MCP wiring.** Both currently resolve `index_db` via the ancestor walk. Extend to also derive `repo_root` (the directory containing `.illuminate/`). Pass it to `Auditor::with_index_and_root` (or whatever shape lands).

**Deliverable.** Commit `feat(audit): normalize absolute file paths against repo root in audit_with_files`.

### Task T — Symbol-level Rust call edges

**File:** `crates/illuminate-index/src/edge_extract.rs`.

**Goal.** Add `extract_rust_call_edges()` alongside `extract_rust_edges` (which handles imports). Walks `call_expression` nodes, extracts the called function's qualified name from the path, emits `Edge { kind: Calls, ... }`.

**Scope reduction for v0.4.x.** Don't try to fully resolve cross-module calls (that's a long pole). Just emit edges where:
- `source_qualified` = the enclosing function's qualified name `<file>::<containing_function>` (file-relative, function-scoped).
- `target_qualified` = the literal text of the call's function path. For `bar()` it's just `"bar"`. For `foo::bar()` it's `"foo::bar"`. For `self.method()` it's `"self.method"`. Don't try to resolve `self`/`crate`/`super` — emit literal text. Future passes can resolve.

**Approach.**
1. Tree-sitter walk for `function_item` nodes (defines a function). For each, recursively walk its body for `call_expression` nodes.
2. For each call, extract the function path (the first child of `call_expression` is typically a `field_expression`, `scoped_identifier`, or `identifier`).
3. Emit one edge per call, with `source_qualified = format!("{}::{}", file_path, fn_name)` and `target_qualified = call_path_text`.

**Update `extract_rust_edges`** to optionally also produce call edges, OR add a new `extract_rust_call_edges` and have `index_file_with_edges` concatenate both vectors. The latter is cleaner.

In `crates/illuminate-index/src/lib.rs::index_file_with_edges`:
```rust
let mut extracted_edges = match lang {
    Language::Rust => edge_extract::extract_rust_edges(&tree, source, &file_path),
    ...
};
if matches!(lang, Language::Rust) {
    extracted_edges.extend(edge_extract::extract_rust_call_edges(&tree, source, &file_path));
}
```

**Tests:**
- `extracts_simple_rust_call` — `fn foo() { bar(); }` → 1 Calls edge `foo::file::lib.rs → bar`. (Adjust expected text to match the actual qualified-name format chosen.)
- `extracts_method_call_rust` — `fn foo() { x.method(); }` → 1 edge target `"x.method"`.
- `extracts_path_qualified_call_rust` — `fn foo() { module::bar(); }` → 1 edge target `"module::bar"`.
- `extracts_nested_calls` — `fn foo() { bar(baz()); }` → 2 edges (one per call site).
- `function_with_no_calls_yields_no_call_edges` — `fn foo() {}` → 0 call edges (but still 0 import edges if no `use`).

**`impact_radius` benefits automatically** — the BFS over edges will now traverse function-call relationships, so seeds expand to calls and callees properly.

**Deliverable.** Commit `feat(index): rust function call edge extraction`.
