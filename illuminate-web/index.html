§ 01 the shift

for thirty years we versioned the right thing. the abstraction just moved.

humans wrote source. compilers produced binaries. git versioned the source because that was the artifact carrying human intent. then AI coding agents arrived. labor moved upward. prompts became the new source — agents now produce the binaries. the industry's tooling is still one layer behind.

humans now write prompts. agents produce source code. the source code is the new binary equivalent. the prompt — and the reasoning that produced it — is the new source.

and yet we still version the binaries. teams commit code to git and discard the prompts that produced it. reasoning evaporates into ~/.claude/projects/<hash>.jsonl files on individual laptops. architectural decisions made on Tuesday are invisible to the agent on Friday.

every AI agent on a team behaves like a brand-new hire who has never read the codebase's history. illuminate is the catch-up — the version control, collaboration, and intelligence layer for the thing humans now actually write.

in 3 years, what teams version, review, and share will not be code. it will be prompts plus the context that produced them.

§ 04 trust model

get this wrong and devs won't install it.

illuminate's value depends on developers trusting it with their unfiltered work. that means three commitments, baked into the architecture, not just the marketing. local-first isn't a feature. it's the only way the compounding-knowledge thesis works.

⌂ stays local · always

your unfiltered work, untouched.

  • ·
    Every session. Captured silently to .illuminate/trail/. Gitignored. Never auto-uploaded.
  • ·
    Personal scratch. Half-formed thoughts, dead ends, debugging spirals, embarrassing prompts — all stay local.
  • ·
    Enrichment queries. Run locally against a local mirror of the team graph. No network call in the prompt path.
  • ·
    Failure context. Captured by reflect. Stays in your trail unless you publish.

no telemetry. no "anonymized usage data." no admin opt-out checkbox you missed.

✓ published · with consent

only what you explicitly ship.

  • ·
    Curated sessions. You choose the format — full, summary, decision-only — and the redaction level. Default is summary; one-keystroke skip.
  • ·
    Decisions & patterns. Surfaced from your published sessions. Written to the team repo as standard markdown.
  • ·
    Failures. Documented through reflect only after you explicitly publish them.
  • ·
    A git repo you own. Standard markdown + json. No proprietary format. Self-hosted or hosted — your choice. Could literally be a github private repo.

if your github account is secure, your illuminate data is secure — same threat model, same controls.

✕ never built

the things we will not ship.

  • ·
    No individual scoring. Illuminate will not produce dashboards rating developers by prompt quality. Period.
  • ·
    No management surveillance. No "see what your team is prompting" admin view. Not now, not later.
  • ·
    No prompt leaderboards. Aggregate team trends only if a team explicitly opts in and the data is genuinely anonymized at source.
  • ·
    No data sale. Your prompts are not training data, not marketing fodder, not a growth lever.

if a competitor builds the surveillance version, that's a different product. the moment illuminate enables it, devs game it, sabotage it, refuse it — and the thesis collapses.

§ 06 architecture · performance

twelve rust crates. one binary. one sqlite file per repo.

built on ctxgraph — a bi-temporal knowledge graph with 2.4× F1 vs Graphiti and ~250× faster ingestion. all extraction runs locally via ONNX (GLiNER + GLiREL + MiniLM). LLM fallback used in ~30% of ingestion paths only — never in queries. enrichment and audit are deterministic, fast, and free at runtime.

crate layout 12 crates · 1 static binary · 0 python · 0 docker
illuminate-core
Graph API on top of ctxgraph. Entity and relationship types, bi-temporal invariants.
illuminate-trail
Session capture daemon. Claude Code, Cursor, Codex — inotify-driven, <1% cpu.
illuminate-enrich
Pre-LLM prompt enrichment — the wedge. Deterministic, local, sub-100ms.
illuminate-publish
Explicit publish gesture. Writes structured markdown + json to the team repo.
illuminate-extract
Local NER pipeline. GLiNER + GLiREL via ONNX. No external API calls.
illuminate-embed
Local embeddings. all-MiniLM-L6-v2 — 90MB model, runs on cpu.
illuminate-index
Code structure indexer. tree-sitter, narrow scope: rust, ts, python, go.
illuminate-audit
Policy engine + drift detection. Cross-references proposed changes against the graph.
illuminate-reflect
Failure capture. Ingests test failures, sentry alerts, manual post-mortems into the graph.
illuminate-route
LLM fallback router. Used in ~30% of ingestion paths — never in queries.
illuminate-mcp
JSON-RPC server. Exposes the audit + query tools to any MCP-aware agent.
illuminate-cli
User-facing binary. init · enrich · publish · browse · audit · reflect

§ 09 scope discipline

what illuminate is not.

the boundary matters as much as the feature set. these are structural commitments, not features awaiting prioritization. if you need them, illuminate is the wrong tool.

×
Not surveillance software.

No individual scoring. No management dashboards rating developers by prompt quality. No prompt leaderboards. If a competitor builds that, it's a different product.

×
Not a replacement for git or GitHub.

Runs alongside them. The team-shared layer can literally live in a GitHub private repo.

×
Not a hosted SaaS by default.

Local-first. Self-hostable. Hosted illuminate.sh is an optional convenience layer added later — not the product.

×
Not prompt management for production APIs.

Langfuse, PromptLayer, Braintrust handle production prompts shipped to end users. Illuminate handles development-time coding sessions — a different artifact.

×
Not a generic AI-powered wiki.

The wiki is a byproduct of the loop, not the product. The product is the prompt-versioning + enrichment system that produces it.

×
Not a code review tool.

Adjacent space, different artifact. Illuminate owns the prompt + reasoning trail, not the review verdict.

×
Not competing with spec-kit.

spec-kit handles work before code is written. Illuminate handles work during and after. They compose — illuminate ingests spec-kit's constitution and specs as decision sources.

×
Not a feature in your IDE.

Runs as a local daemon and CLI. Editor extensions are thin clients over the same surface. The substrate is the source of truth.
